# Learning Objectives

## Overview

Project Aegis is designed to build practical cybersecurity engineering skills through a structured, hands-on homelab.

The project combines technical implementation, security analysis, documentation, validation, and professional reporting. Each milestone builds on the previous one so that the environment gradually develops into a small enterprise-style security lab.

## Foundation Objectives

By completing the foundation phase, I will be able to:

* Explain the purpose of virtualization
* Describe the difference between Type 1 and Type 2 hypervisors
* Explain how Proxmox uses KVM and LXC
* Create, configure, start, stop, and delete virtual machines
* Allocate CPU, memory, storage, and network resources
* Use snapshots before significant configuration changes
* Organize virtual machines and documentation consistently
* Use Git and GitHub to track project changes
* Write clear Markdown documentation
* Maintain an asset inventory

## Linux Objectives

By completing the Linux phase, I will be able to:

* Install and configure Ubuntu Server
* Navigate the Linux filesystem
* Manage files and directories
* Create and manage users and groups
* Understand Linux file permissions
* Install and update software packages
* Manage services with `systemctl`
* Review active processes and network connections
* Configure SSH access
* Locate and interpret basic Linux logs
* Apply basic Linux hardening practices

## Windows Objectives

By completing the Windows phase, I will be able to:

* Install and configure a Windows workstation
* Manage local users and groups
* Apply operating-system updates
* Configure Windows Defender Firewall
* Review running processes and services
* Use Event Viewer
* Use PowerShell for basic administration
* Review network configuration
* Configure endpoint logging
* Apply basic Windows hardening practices

## Networking Objectives

By completing the networking phase, I will be able to:

* Explain IP addresses, subnet masks, gateways, and DNS
* Explain the difference between TCP and UDP
* Identify common ports and protocols
* Understand switching, routing, NAT, and DHCP
* Configure virtual network interfaces
* Test connectivity with `ping`, `traceroute`, and related tools
* Identify open ports and listening services
* Document network flows and trust boundaries
* Create basic network diagrams
* Explain how segmentation reduces risk

## Vulnerability Management Objectives

By completing the vulnerability management phase, I will be able to:

* Perform host discovery
* Enumerate ports and services with Nmap
* Explain the difference between discovery, enumeration, and vulnerability scanning
* Deploy and configure Greenbone/OpenVAS
* Run authenticated and unauthenticated scans
* Interpret vulnerability scan results
* Understand CVEs and CVSS scores
* Prioritize findings using severity, exposure, exploitability, and impact
* Create a remediation plan
* Apply patches and configuration changes
* Conduct validation scans
* Compare before-and-after results
* Write a vulnerability assessment report

## Identity and Access Management Objectives

By completing the identity phase, I will be able to:

* Explain the purpose of centralized identity management
* Deploy Windows Server
* Configure Active Directory Domain Services
* Create organizational units, users, groups, and computers
* Join systems to a domain
* Configure basic Group Policy
* Explain authentication and authorization
* Apply the principle of least privilege
* Identify common identity and privilege risks
* Review administrative access
* Document identity-related controls

## Network Security Objectives

By completing the network security phase, I will be able to:

* Deploy and configure pfSense
* Create network segments
* Configure routing between networks
* Create and test firewall rules
* Configure NAT, DHCP, and DNS services
* Restrict administrative access
* Document permitted network flows
* Identify unnecessary exposure
* Explain default-deny firewall design
* Validate segmentation controls

## Security Monitoring Objectives

By completing the monitoring phase, I will be able to:

* Explain the purpose of centralized logging

* Collect Windows and Linux logs
* Configure Sysmon

* Create dashboards

By completing the detection engineering phase, I will be able to:

* Validate that detections trigger correctly

## Final Outcome

At the end of Project Aegis, I should be able to demonstrate that I can:

1. Build a small enterprise-style environment
2. Secure and monitor its systems
3. Identify and prioritize vulnerabilities
4. Implement and validate security controls
5. Investigate suspicious activity
6. Document technical decisions
7. Communicate risk clearly
8. Explain how the environment supports business and security objectives
