# Project Overview

## Project Name

**Project Aegis**

## Purpose

Project Aegis is a hands-on cybersecurity homelab designed to develop practical skills in infrastructure security, vulnerability management, identity security, network defense, centralized logging, detection engineering, threat hunting, and security architecture.

The lab is hosted on Proxmox and will gradually develop into a small enterprise-style environment containing Linux servers, Windows workstations, Active Directory, segmented networks, security monitoring, vulnerability scanning, and cloud resources.

The project emphasizes both technical implementation and professional documentation.

## Primary Goal

The primary goal is to demonstrate the ability to design, build, secure, assess, monitor, and document an enterprise-style technology environment.

Each project phase will include:

* Technical implementation
* Security analysis
* Validation and testing
* Architecture diagrams
* Configuration documentation
* Troubleshooting notes
* Risk and control analysis
* GitHub updates
* Portfolio-ready reports

## Career Alignment

Project Aegis supports career development toward the following roles:

* Security Engineer
* Infrastructure Security Engineer
* Cloud Security Engineer
* Identity and Access Management Engineer
* Security Architect
* Infrastructure Security Architect

The project builds on existing infrastructure and solution-design experience while developing direct security ownership in areas such as vulnerability management, logging, hardening, access control, segmentation, detection, and risk-based architecture.

## Guiding Principles

### Understand Before Implementing

Commands and configuration changes should not be applied without understanding their purpose, expected behavior, and potential risks.

### Document the Process

Documentation is treated as part of the technical work rather than an activity completed afterward.

### Validate Every Change

Security controls and configuration changes must be tested to confirm that they work as intended.

### Use Risk-Based Decisions

Findings and improvements will be prioritized according to exposure, exploitability, impact, and remediation effort.

### Protect Sensitive Information

Passwords, credentials, public IP addresses, private keys, employer information, and unsanitized security results must not be committed to GitHub.

### Build Incrementally

The environment will be developed in controlled milestones so that each component can be understood, tested, documented, and maintained.

## Lab Platform

The lab runs on an ASUS TUF Gaming Laptop FX504 with:

* Intel Core i7-8750H processor
* 6 physical cores and 12 threads
* 16 GB DDR4 memory
* 256 GB PCIe SSD
* 1 TB HDD
* Proxmox Virtual Environment

Because memory is limited, only the virtual machines required for the current activity will normally remain powered on.

## Definition of Success

Project Aegis will be considered successful when the environment can be built, secured, monitored, assessed, and explained through clear documentation.

A successful final portfolio should allow a technical reviewer to understand:

1. What was built
2. Why it was designed that way
3. Which risks were identified
4. Which controls were implemented
5. How those controls were validated
6. What problems were encountered
7. What was learned
8. How the project relates to professional security work
